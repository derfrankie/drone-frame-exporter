# Notes

- UI comes after the CLI MVP.
- Planned desktop preview path is `PySide6.QtMultimedia`.
- Frame extraction and metadata writing stay delegated to external tools for reproducibility.
- Visual direction should follow `https://onyx.scharz`.
- The user workflow is "pick individual best photos visually", not "enter timestamps manually" as the main UX.
- The UI needs a simple map-based way to place the current frame along the GPX track.
- The sync UX needs both a fine-grained second offset and a coarser hour-shift control for timezone mismatches.
